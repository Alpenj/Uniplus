# 개발 가이드 (Developer Guide)

앞으로 이 프로젝트를 계속 수정하고 관리하는 방법이다.
복잡하게 생각할 거 없다. 딱 3단계만 기억해라.

## 1. 개발 시작 (Local Development)
컴퓨터에서 작업할 때 켜야 하는 명령어다.
```bash
# 개발 서버 실행 (터미널 1)
npm run dev -- -H 0.0.0.0

# (선택) 외부에서 폰으로 테스트할 때 (터미널 2)
npx localtunnel --port 3000
```

## 2. 코드 수정 & 저장 (Git Workflow)
열심히 코드를 짰으면 깃허브에 올려야 한다. (Vercel 배포도 이걸로 자동 해결됨)

```bash
# 1. 변경된 파일들 장바구니에 담기
git add .

# 2. 꼬리표 붙여서 포장하기 (메시지는 수정 내용대로 적으셈)
git commit -m "수정: 메인 페이지 폰트 크기 변경"

# 3. 깃허브로 쏘기 (이거 하면 Vercel이 알아서 배포 시작함)
git push
```

## 3. 배포 확인 (Production)
`git push` 하고 1~2분 뒤에 들어가면 바뀐 내용 적용되어 있다.
- **주소**: https://uniplus-rust.vercel.app/

---

### 🚨 주의사항
- **강제 푸시 금지**: 다른 사람이랑 같이 하는 거 아니면 `git push -f` 같은 건 웬만하면 쓰지 마라. (아까는 정리하느라 한 거임)
- **에러 나면**: `git status` 쳐보면 현재 상태 알려주니까 그거 보고 해결해라.

## 4. 심화: 커밋 정리하기 (Git Squash 상세 가이드)

다른 프로젝트에서도 쓸 수 있는 공통적인 방법이다.
**상황**: "기능 A를 만드느라 커밋을 3개나 지저분하게 찍었는데, 이걸 1개로 깔끔하게 합쳐서 올리고 싶다."

### 1단계: 몇 개 합칠지 확인하기
가장 먼저 내가 작업한 내역을 확인해서 **"몇 개(N)"**를 되돌릴지 센다.
```bash
git log --oneline
# (화면에 나오는 거 보고 개수 세기)
```

### 2단계: 시간을 되돌리기 (Soft Reset)
`--soft` 옵션이 핵심이다. "**커밋 기록만 지우고, 파일 수정 내용은 그대로 둬라**"라는 뜻이다.
```bash
# 예: 3개를 합치고 싶으면 3
git reset --soft HEAD~3
```
이걸 입력하면 방금 했던 3번의 커밋은 사라지지만, 작업한 파일들은 "아직 커밋 안 된 상태"로 안전하게 남는다.

### 3단계: 예쁘게 다시 포장하기 (Re-commit)
이제 모아둔 변경 사항들을 하나의 깔끔한 메시지로 다시 묶는다.
```bash
git commit -m "깔끔하게 정리된 메시지"
```

### 4단계: 덮어씌우기 (Force Push)
**주의**: 이미 깃허브(원격)에 지저분한 커밋들이 올라가 있었다면, 그냥 `push` 하면 에러 난다. (역사가 달라졌기 때문)
이때는 "내가 정리한 게 맞으니까 이걸로 덮어씌워!"라고 강제 명령을 내려야 한다.
```bash
git push -f
```
*(혼자 쓰는 프로젝트에서는 괜찮지만, 팀 프로젝트 공유 브랜치에서는 쓰면 안 됨)*
